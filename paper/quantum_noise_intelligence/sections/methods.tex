\section{Methods}
\label{sec:methods}

\subsection{Physically-Bounded ZNE with AICc Selection}
\label{sec:bounded_zne}

\subsubsection{Problem formulation}

Given noise-amplified measurements $\{(\lambda_i, y_i)\}_{i=1}^n$ where $\lambda_i$ are noise scale factors and $y_i = \langle O \rangle_{\lambda_i}$ are measured expectation values, ZNE seeks to estimate $\langle O \rangle_0 = f(0;\theta)$ for some parametric model $f$.

We fit three candidate models, each representing a physically motivated noise decay:
\begin{align}
    f_\text{poly}(\lambda;\theta) &= a_0 + a_1\lambda + a_2\lambda^2, \quad k=3 \label{eq:poly} \\
    f_\text{exp}(\lambda;\theta) &= a + b\,e^{-c\lambda}, \quad c > 0, \quad k=3 \label{eq:exp} \\
    f_\text{polyexp}(\lambda;\theta) &= (a_0 + a_1\lambda)\,e^{-c\lambda}, \quad k=4 \label{eq:polyexp}
\end{align}

The polynomial model captures algebraic decay typical of depolarizing noise; the exponential model captures the exponential fidelity decay under Markovian noise channels; the poly-exponential model accommodates circuits where different gate layers experience different effective noise rates.

\subsubsection{Constrained optimization}

For each model, we solve:
\begin{equation}
    \hat{\theta} = \arg\min_\theta \sum_{i=1}^{n} \left[ f(\lambda_i; \theta) - y_i \right]^2 \quad \text{s.t.} \quad l_b \leq f(0; \theta) \leq u_b,
    \label{eq:bounded_opt}
\end{equation}
where $[l_b, u_b]$ are the spectral bounds of the observable ($[0,1]$ for probabilities, $[-1,1]$ for Pauli expectations). We solve this using L-BFGS-B~\cite{Byrd1995} with the bound constraint applied directly to the parameter combination that determines $f(0;\theta)$:
\begin{itemize}
    \item Polynomial: $f(0) = a_0$, so $l_b \leq a_0 \leq u_b$
    \item Exponential: $f(0) = a + b$, so $l_b \leq a + b \leq u_b$
    \item Poly-exponential: $f(0) = a_0$, so $l_b \leq a_0 \leq u_b$
\end{itemize}

Initial parameters are estimated from data: $a_0 \approx y_1$, $c \approx -\ln(y_n/y_1)/(\lambda_n - \lambda_1)$, ensuring the optimizer starts in a physically reasonable region.

\subsubsection{AICc model selection}

Given $n$ data points and a model with $k$ parameters and residual sum of squares $\text{RSS} = \sum_i [f(\lambda_i;\hat\theta) - y_i]^2$, the corrected Akaike Information Criterion is:
\begin{equation}
    \text{AICc} = n\ln\!\left(\frac{\text{RSS}}{n}\right) + 2k + \frac{2k(k+1)}{n-k-1}.
    \label{eq:aicc}
\end{equation}

The model with minimum AICc is selected. The correction term $2k(k+1)/(n-k-1)$ is critical for ZNE:

\caption{AICc penalty for different model complexities at typical ZNE sample sizes. The correction strongly penalizes the 4-parameter model when $n \leq 7$.}
\label{tab:aicc_penalty}
\begin{ruledtabular}

\hline

When $n=5$ (our default), the 4-parameter poly-exponential model receives a penalty of $2 \times 4 + 2 \times 4 \times 5 / (5-4-1) = \infty$ (undefined---the model is automatically excluded). With $n=5$ and $k=3$, the penalty is $6 + 12 = 18$ versus $4 + 6 = 10$ for $k=2$, requiring the 3-parameter model to explain 8 additional units of log-likelihood to be preferred. This provides strong regularization against overfitting.

\subsubsection{Theoretical justification}

The combination of bounds and AICc addresses complementary failure modes. We formalize this with two results.

\textbf{Theorem 1} (Physical validity guarantee). \textit{Let $f(\lambda;\theta)$ be any parametric model from the candidate set $\{f_\text{poly}, f_\text{exp}, f_\text{polyexp}\}$, and let $\hat\theta$ be the solution to the constrained optimization problem~(\ref{eq:bounded_opt}). Then for any input data $\{(\lambda_i, y_i)\}_{i=1}^n$ with $y_i \in \mathbb{R}$, the zero-noise estimate satisfies $l_b \leq f(0;\hat\theta) \leq u_b$.}

\textit{Proof.} The L-BFGS-B algorithm solves~(\ref{eq:bounded_opt}) subject to box constraints on the parameter subspace determining $f(0;\theta)$. Specifically:
\begin{itemize}
    \item For $f_\text{poly}$: $f(0) = a_0$, constrained as $l_b \leq a_0 \leq u_b$.
    \item For $f_\text{exp}$: $f(0) = a + b$. We reparametrize $s = a + b$ and constrain $l_b \leq s \leq u_b$.
    \item For $f_\text{polyexp}$: $f(0) = a_0$, constrained as $l_b \leq a_0 \leq u_b$.
\end{itemize}
In each case, the feasible set $\Theta_\text{feas} = \{\theta : l_b \leq f(0;\theta) \leq u_b\}$ is non-empty (taking $f(0;\theta) = (l_b + u_b)/2$ is always feasible) and closed. Since L-BFGS-B returns a point in the feasible set (or the projection onto it), the bound is satisfied by construction. The constraint is active (binding) only when the unconstrained optimum violates the bound, in which case the estimate is clipped to $l_b$ or $u_b$. $\square$

\textbf{Theorem 2} (AICc prevents overfitting in the ZNE regime). \textit{Let $\mathcal{M} = \{M_1, \ldots, M_m\}$ be a set of candidate models with $k_j$ parameters for model $M_j$. Assume Gaussian residuals with unknown variance. For $n$ data points with $n - k_j - 1 > 0$, AICc satisfies:}
\begin{equation}
    \text{AICc}(M_j) = n\ln\!\left(\frac{\text{RSS}_j}{n}\right) + 2k_j + \frac{2k_j(k_j+1)}{n - k_j - 1}.
\end{equation}
\textit{The correction term $C(k,n) = 2k(k+1)/(n-k-1)$ has the following properties:}
\begin{enumerate}
    \item[(i)] \textit{$C(k,n) \to \infty$ as $k \to n-1$ (models approaching saturation are excluded);}
    \item[(ii)] \textit{$C(k,n) \to 0$ as $n \to \infty$ (AICc reduces to AIC asymptotically);}
    \item[(iii)] \textit{For $n=5$: $C(2,5) = 3.0$, $C(3,5) = 12.0$, $C(4,5) = \infty$, providing strong discrimination against complex models.}
\end{enumerate}

\textit{Proof.} Properties (i)--(iii) follow directly from the functional form of $C(k,n)$. For (i): as $k \to n-1$, the denominator $n-k-1 \to 0^+$, so $C \to \infty$. For (ii): for fixed $k$, $C(k,n) = 2k(k+1)/(n-k-1) \sim 2k(k+1)/n \to 0$. For (iii): direct substitution.

The practical consequence is that with $n=5$ noise scale factors (our default), the 4-parameter poly-exponential model is automatically excluded ($C = \infty$), and the 3-parameter models must explain $C(3,5) - C(2,5) = 9.0$ additional units of negative log-likelihood relative to the 2-parameter linear model to be selected. This corresponds to requiring the RSS ratio $\text{RSS}_\text{linear}/\text{RSS}_\text{complex} > e^{9/5} \approx 6.0$---a stringent requirement that prevents selection of complex models unless they provide substantially better fit.

Hurvich and Tsai~\cite{Hurvich1989} proved that AICc is asymptotically efficient (selects the best approximating model with probability approaching 1) and provides a second-order bias correction relative to AIC. Burnham and Anderson~\cite{Burnham2002} established that AICc outperforms BIC for model selection when the true model is not in the candidate set---a realistic assumption for ZNE where the true noise decay may not exactly match any candidate. $\square$

\textbf{Corollary} (Synergy of bounds + AICc). \textit{Bounds alone prevent unphysical predictions but do not prevent inaccurate predictions within $[l_b, u_b]$. AICc alone prevents overfitting but does not prevent unphysical predictions from well-selected models. The combination eliminates both failure modes simultaneously.}

This is demonstrated empirically: bounded optimization with RSS selection achieves 0\% unphysical rate but only 29\% win rate (inaccurate within bounds due to overfitting). Adding AICc raises the win rate to 61.6\% while maintaining 0\% unphysical rate.

\subsection{AutoMitigator: Intelligent Strategy Selection}
\label{sec:automitigator}

Not all circuits benefit equally from ZNE. We implement \textsc{AutoMitigator}, a rule-based strategy selector that analyzes circuit properties and noise characteristics:

\begin{figure}[!htbp]
\hrule\vspace{0.5em}
\textbf{Algorithm 1:} AutoMitigator Strategy Selection\\[0.3em]
\textbf{Input:} Circuit $C$ (depth $d$, 2Q gates $g_{2q}$, qubits $q$); Noise ($\epsilon_g$, $\epsilon_r$)\\
\textbf{Output:} Strategy $s$\\[0.3em]
1. \textbf{if} $d < 5$ and $g_{2q} < 3$: return NONE\\
2. \textbf{if} $\epsilon_r > 3\epsilon_g$: return READOUT\\
3. \textbf{if} $g_{2q}/q > 5$ and $\epsilon_g > 0.01$: return BOUNDED-ZNE\\
4. \textbf{if} $d > 20$ and $q > 4$: return COMBINED\\
5. \textbf{default}: return BOUNDED-ZNE
\vspace{0.5em}\hrule
\caption{AutoMitigator strategy selection. The algorithm routes circuits to the most appropriate mitigation technique based on structural and noise properties.}
\label{alg:automitigator}
\end{figure}

The selector outputs a recommended strategy with confidence score and human-readable justification, enabling both automated pipelines and interactive use.

\subsection{MCP Server Architecture}
\label{sec:mcp}

\begin{figure}[!htbp]
    \includegraphics[width=\columnwidth]{../figures/fig_architecture.pdf}
    \caption{System architecture. AI agents interact via the Model Context Protocol (top), which routes requests to the physics-informed mitigation engine (middle), executing on quantum backends (bottom). Physical bounds and AICc selection are embedded in the tools, providing guardrails regardless of how agents invoke them.}
    \label{fig:architecture}
\end{figure}

We implement a Model Context Protocol~\cite{Anthropic2024} server exposing eight tools for quantum error mitigation via the FastMCP framework (stdio transport). The server architecture (\cref{fig:architecture}) addresses a key accessibility gap: while tools like Mitiq~\cite{LaRose2022} provide powerful QEM primitives, they require quantum computing expertise to use correctly. Our MCP server encapsulates domain knowledge---noise characterization, strategy selection, physical bounds---behind natural language interfaces.

The eight exposed tools are: \texttt{noise\_profile} (backend characterization), \texttt{recommend\_mitigation} (strategy selection via AutoMitigator), \texttt{run\_bounded\_zne} (our core algorithm), \texttt{compare\_strategies} (multi-method benchmarking), \texttt{optimize\_circuit} (noise-aware compilation), \texttt{get\_calibration\_data} (hardware data retrieval), \texttt{wukong\_status} (hardware availability), and \texttt{run\_experiment} (end-to-end pipeline).

Server startup time is 1.7 seconds (lazy-loading heavy dependencies), and tool response time averages under 3 seconds for bounded ZNE computation on typical circuits (2--6 qubits, depth $\leq 30$).

\subsection{Noise Model and Simulation Platform}
\label{sec:noise_model}

All benchmarks use Qiskit Aer~\cite{Qiskit2024} with noise models calibrated from real Origin Wukong 180 hardware data (169 qubits, 396 CZ gates, calibration data extracted via OriginQ Cloud API):
\begin{itemize}
    \item Depolarizing noise: $\epsilon_\text{1Q} = 0.5\%$, $\epsilon_\text{CZ} \in \{1\%, 3\%, 5\%, 8\%, 12\%\}$
    \item Symmetric readout error: $p(0|1) = p(1|0) = 2\%$
    \item Shot count: 4096 per circuit execution
    \item Noise amplification: unitary folding~\cite{GiurgicaTiron2020} with $\lambda \in \{1, 1.5, 2, 2.5, 3\}$
\end{itemize}

Hardware validation on Origin Wukong 180 is pending resolution of an API compatibility issue. The noise parameters above are derived from real calibration data, providing a realistic (though not identical) approximation of hardware behavior.
