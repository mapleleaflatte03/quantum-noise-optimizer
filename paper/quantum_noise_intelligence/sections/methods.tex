\section{Methods}
\label{sec:methods}

\subsection{Physically-Bounded ZNE with AICc Selection}
\label{sec:bounded_zne}

\subsubsection{Problem formulation}

Given noise-amplified measurements $\{(\lambda_i, y_i)\}_{i=1}^n$ where $\lambda_i$ are noise scale factors and $y_i = \langle O \rangle_{\lambda_i}$ are measured expectation values, ZNE seeks to estimate $\langle O \rangle_0 = f(0;\theta)$ for some parametric model $f$.

We fit three candidate models, each representing a physically motivated noise decay:
\begin{align}
    f_\text{poly}(\lambda;\theta) &= a_0 + a_1\lambda + a_2\lambda^2, \quad k=3 \label{eq:poly} \\
    f_\text{exp}(\lambda;\theta) &= a + b\,e^{-c\lambda}, \quad c > 0, \quad k=3 \label{eq:exp} \\
    f_\text{polyexp}(\lambda;\theta) &= (a_0 + a_1\lambda)\,e^{-c\lambda}, \quad k=4 \label{eq:polyexp}
\end{align}

The polynomial model captures algebraic decay typical of depolarizing noise; the exponential model captures the exponential fidelity decay under Markovian noise channels; the poly-exponential model accommodates circuits where different gate layers experience different effective noise rates.

\subsubsection{Constrained optimization}

For each model, we solve:
\begin{equation}
    \hat{\theta} = \arg\min_\theta \sum_{i=1}^{n} \left[ f(\lambda_i; \theta) - y_i \right]^2 \quad \text{s.t.} \quad l_b \leq f(0; \theta) \leq u_b,
    \label{eq:bounded_opt}
\end{equation}
where $[l_b, u_b]$ are the spectral bounds of the observable ($[0,1]$ for probabilities, $[-1,1]$ for Pauli expectations). We solve this using L-BFGS-B~\cite{Byrd1995} with the bound constraint applied directly to the parameter combination that determines $f(0;\theta)$:
\begin{itemize}
    \item Polynomial: $f(0) = a_0$, so $l_b \leq a_0 \leq u_b$
    \item Exponential: $f(0) = a + b$, so $l_b \leq a + b \leq u_b$
    \item Poly-exponential: $f(0) = a_0$, so $l_b \leq a_0 \leq u_b$
\end{itemize}

Initial parameters are estimated from data: $a_0 \approx y_1$, $c \approx -\ln(y_n/y_1)/(\lambda_n - \lambda_1)$, ensuring the optimizer starts in a physically reasonable region.

\subsubsection{AICc model selection}

Given $n$ data points and a model with $k$ parameters and residual sum of squares $\text{RSS} = \sum_i [f(\lambda_i;\hat\theta) - y_i]^2$, the corrected Akaike Information Criterion is:
\begin{equation}
    \text{AICc} = n\ln\!\left(\frac{\text{RSS}}{n}\right) + 2k + \frac{2k(k+1)}{n-k-1}.
    \label{eq:aicc}
\end{equation}

The model with minimum AICc is selected. The correction term $2k(k+1)/(n-k-1)$ is critical for ZNE:

When $n=5$ (our default), the 4-parameter poly-exponential model receives a penalty of $2 \times 4 + 2 \times 4 \times 5 / (5-4-1) = \infty$ (undefined---the model is automatically excluded). With $n=5$ and $k=3$, the penalty is $6 + 12 = 18$ versus $4 + 6 = 10$ for $k=2$, requiring the 3-parameter model to explain 8 additional units of log-likelihood to be preferred. This provides strong regularization against overfitting.

\subsubsection{Theoretical justification}

The combination of bounds and AICc addresses complementary failure modes:

\textbf{Proposition 1} (Bounds eliminate unphysical predictions). \textit{For any model $f$ and any data $\{y_i\}$, the constrained estimate satisfies $l_b \leq f(0;\hat\theta) \leq u_b$ by construction.}

This is trivially guaranteed by the optimization constraint. The non-trivial question is whether the bounded estimate is \emph{accurate}.

\textbf{Proposition 2} (AICc consistency in the ZNE regime). \textit{Under regularity conditions, AICc is asymptotically efficient for model selection~\cite{Hurvich1989}. In the finite-sample regime $n/k < 40$, AICc provides a second-order bias correction that prevents selection of overparameterized models.}

The practical consequence: when the true noise decay is approximately linear (as at low noise), AICc correctly selects the 2-parameter linear model over the 3-parameter polynomial, avoiding the overfitting that causes bounded-but-inaccurate estimates. When the true decay is genuinely nonlinear (high noise, deep circuits), AICc permits selection of the exponential model.

\textbf{Failure mode analysis.} Without AICc (using RSS alone), the 3-parameter polynomial always achieves lower RSS than the 2-parameter linear model on $n=5$ points, regardless of the true underlying function. This leads to systematic overfitting: our initial implementation using RSS selection achieved only 29\% win rate over linear ZNE. After switching to AICc, win rate improved to 61.6\%---a direct consequence of appropriate model complexity selection.

\subsection{AutoMitigator: Intelligent Strategy Selection}
\label{sec:automitigator}

Not all circuits benefit equally from ZNE. We implement \textsc{AutoMitigator}, a rule-based strategy selector that analyzes circuit properties and noise characteristics:

\textbf{Input:} Circuit $C$ (depth $d$, 2Q gates $g_{2q}$, qubits $q$); Noise ($\epsilon_g$, $\epsilon_r$)\\
\textbf{Output:} Strategy $s$\\[0.3em]

The selector outputs a recommended strategy with confidence score and human-readable justification, enabling both automated pipelines and interactive use.

\subsection{MCP Server Architecture}
\label{sec:mcp}

We implement a Model Context Protocol~\cite{Anthropic2024} server exposing eight tools for quantum error mitigation via the FastMCP framework (stdio transport). The server architecture addresses a key accessibility gap: while tools like Mitiq~\cite{LaRose2022} provide powerful QEM primitives, they require quantum computing expertise to use correctly. Our MCP server encapsulates domain knowledge---noise characterization, strategy selection, physical bounds---behind natural language interfaces.

The eight exposed tools are: \texttt{noise\_profile} (backend characterization), \texttt{recommend\_mitigation} (strategy selection via AutoMitigator), \texttt{run\_bounded\_zne} (our core algorithm), \texttt{compare\_strategies} (multi-method benchmarking), \texttt{optimize\_circuit} (noise-aware compilation), \texttt{get\_calibration\_data} (hardware data retrieval), \texttt{wukong\_status} (hardware availability), and \texttt{run\_experiment} (end-to-end pipeline).

Server startup time is 1.7 seconds (lazy-loading heavy dependencies), and tool response time averages under 3 seconds for bounded ZNE computation on typical circuits (2--6 qubits, depth $\leq 30$).

\subsection{Noise Model and Simulation Platform}
\label{sec:noise_model}

All benchmarks use Qiskit Aer~\cite{Qiskit2024} with noise models calibrated from real Origin Wukong 180 hardware data (169 qubits, 396 CZ gates, calibration data extracted via OriginQ Cloud API):
\begin{itemize}
    \item Depolarizing noise: $\epsilon_\text{1Q} = 0.5\%$, $\epsilon_\text{CZ} \in \{1\%, 3\%, 5\%, 8\%, 12\%\}$
    \item Symmetric readout error: $p(0|1) = p(1|0) = 2\%$
    \item Shot count: 4096 per circuit execution
    \item Noise amplification: unitary folding~\cite{GiurgicaTiron2020} with $\lambda \in \{1, 1.5, 2, 2.5, 3\}$
\end{itemize}

Hardware validation on Origin Wukong 180 is pending resolution of an API compatibility issue. The noise parameters above are derived from real calibration data, providing a realistic (though not identical) approximation of hardware behavior.
